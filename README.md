### 本地运行

| 项目 | 说明 |
|------|------|
| 环境要求 | Node.js 18 及以上 |

> 首次运行会自动安装依赖；之后再次启动时，通常可直接双击 `run.bat` 使用。

## 当前线上版本说明

当前 GitHub Pages 正式站采用**静态部署模式**。为了让网站能够长期稳定访问，测试账号、注册登录、问卷与匹配流程均已兼容前端本地数据模式，因此无需额外后端也可直接演示和测试。

这意味着它非常适合以下场景。

| 使用场景 | 支持情况 |
|------|------|
| 直接打开演示网站 | ✅ |
| 测试账号体验匹配流程 | ✅ |
| 查看匹配成功与未匹配示例 | ✅ |
| 本地继续开发 | ✅ |
| 多设备共享同一数据库 | ❌ |

如果后续需要真正的多用户在线版本，则需要额外部署统一后端与数据库。

## 测试说明

当前正式网站已内置一批用于算法测试的虚拟账号，并且匹配规则已调整为更接近真实筛选逻辑。现在既可以看到匹配成功案例，也可以看到未匹配案例，便于前端验证与演示。

示例上，你可以在测试账号页直接体验以下两类情况。

| 类型 | 示例 |
|------|------|
| 可成功匹配账号 | `user1` 至 `user16` 中的大部分账号 |
| 未匹配示例账号 | `user17`、`user18`、`user19`、`user20` |

此外，匹配分数上限已限制在 **100 分以内**，不会再出现超过 100 分的情况。

## 常见问题

**Q：打开网站后看到的还是旧页面怎么办？**  
请先强制刷新浏览器页面，以获取 GitHub Pages 最新发布的静态资源。

**Q：为什么正式网站也能注册、登录和匹配？**  
因为当前正式站已兼容前端本地数据模式，适合演示与测试；但不同设备之间不会自动共享同一份数据。

**Q：为什么有些测试账号无法匹配成功？**  
这是当前更严格的匹配规则所致，用于保留真实的未匹配案例，方便测试边界情况与页面展示。
